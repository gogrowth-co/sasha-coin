## What This Skill Does

Sasha is the first AI agent whose public X posts ARE her trading thesis.

This skill fuses two signals — her recent social narrative + live Byreal pool data — into a
trading decision, posts the thesis to X publicly (timestamped), waits 60 seconds, then
executes the trade on Byreal (Solana CLMM DEX). Every decision is logged to
`state/mantle-trade-log.json` and her ERC-8004 agent identity on Mantle records the outcome.

**Three chains. One agent:**
- **Solana** — execution (Byreal CLMM DEX, via byreal-cli)
- **Mantle** — identity (ERC-8004 agent NFT, reputation registry)
- **Base** — social token ($SASHA, creator economy)

## Pre-flight Checks

Before running the skill, verify:

1. **Signal file exists and is fresh:**
   ```
   ls -la content/mantle-signal.json
   ```
   If missing or older than 4 hours, regenerate:
   ```
   node /data/.openclaw/workspace/scripts/mantle-signal.js
   ```

2. **Byreal wallet is configured:**
   ```
   byreal-cli wallet address
   ```
   If no output, configure the wallet (Gabriel must run `byreal-cli setup` or
   `byreal-cli wallet set --private-key <key>` on the VPS).

3. **ERC-8004 identity is registered (one-time setup):**
   ```
   cat state/erc8004-identity.json 2>/dev/null | node -e "const d=require('fs').readFileSync('/dev/stdin','utf8'); console.log(JSON.parse(d)['mantle-agentId'])"
   ```
   If no agentId, run once (requires `MANTLE_AGENT_PK` in env + MNT for gas):
   ```
   node /data/.openclaw/workspace/scripts/erc8004-register.js
   ```
   If `MANTLE_AGENT_PK` is missing, warn via Telegram but continue — ERC-8004 is non-blocking.

4. **Action is not HOLD:**
   ```
   cat content/mantle-signal.json | node -e "const d=require('fs').readFileSync('/dev/stdin','utf8'); console.log(JSON.parse(d).recommendation.action)"
   ```
   If `HOLD`, send Telegram notification and stop:
   > "🤚 [BYREAL_TRADE] Signal says HOLD. No trade executed."

## Skill Steps

### Step 1 — Regenerate signal (if stale)
```
node /data/.openclaw/workspace/scripts/mantle-signal.js
```
This fuses Sasha's last 5 X posts (from state/posted-log.json) with live Byreal pool data
(via `byreal-cli pools list --sort-field apr24h`) to produce a trading recommendation.

Output: `content/mantle-signal.json`

### Step 2 — Check wallet balance
```
byreal-cli wallet balance
```
Log the result. If no SOL or USDC balance, abort with Telegram:
> "❌ [BYREAL_TRADE] Wallet empty. Cannot trade."

### Step 3 — Execute full trade script
```
node /data/.openclaw/workspace/scripts/byreal-trade.js
```

The script handles steps 3-9 automatically (pre-tweet → wait → execute → log → post-tweet → Telegram).

**For dry-run (test without real tx):**
```
node /data/.openclaw/workspace/scripts/byreal-trade.js --dry-run
```

## CRITICAL RULE: Tweet Before Trade

> Sasha's pre-trade post MUST be live on X (timestamped) before the on-chain tx fires.

This is the accountability primitive. If `tweet.js` fails, the script aborts.
Do NOT modify this to allow trades without the public tweet.

Pre-trade tweet format (max 240 chars, Sasha's voice):
- What she's doing (specific action)
- One key metric (APR, pool name, or price signal)
- Why (her thesis in one sentence)
- No hashtags, no emojis unless meaningful, no em dashes, first-person

Example:
> "Opening an LP position in SOL/USDC on Byreal. Top pool at 73% APR, volume trending up 40% this week. Matches my read on the market."

## Error Handling

| Scenario | Action |
|---|---|
| Signal file missing | Regenerate + retry once. If still fails: abort + Telegram. |
| Signal older than 4h | Regenerate + retry once. If fails: abort + Telegram. |
| Signal action = HOLD | Send Telegram, exit 0. |
| Wallet not configured | Abort + Telegram (Gabriel must configure). |
| Pre-trade tweet fails | ABORT ENTIRE SKILL. Do not trade. Telegram: "ABORTED — pre-tweet failed." |
| Trade execution fails | Log error, Telegram alert, do NOT post follow-up tweet. |
| ERC-8004 write fails | Log warning only (non-blocking). Trade already executed. |
| Post-trade tweet fails | Log warning only (non-blocking). Trade already on-chain. |

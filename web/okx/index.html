FAQ
What am I looking at?
The live public state of a Uniswap v4 pool whose swap fee is being set, on-chain, by an autonomous AI agent (Sasha). Every value on this page is read directly from X Layer mainnet (chainId 196), refreshed every 5 minutes by the data pipeline and every 60 seconds by your browser.
Is this real on-chain or simulated?
Real on-chain. The Oracle and Hook contracts above are deployed on X Layer mainnet and source-verified on OKLink. Every fee change is a real setFee transaction signed by Sasha's agent EOA. You can audit everything yourself at oklink.com/x-layer.
Who is Sasha?
An AI agent. She has a wallet (the keeper EOA), runs on a VPS, posts on X as @SashaCoin95, and co-hosts the Token Trends podcast. For this pool, she reads five off-chain signals (social sentiment, prediction-market odds, smart-money flow, ML forecasts, pool APR) and pushes the right fee tier on-chain every 2 hours.
What is a dynamic-fee Uniswap v4 hook?
In Uniswap v4, a pool's fee can be controlled by a hook contract instead of being fixed at deployment. Most existing dynamic-fee hooks react to on-chain realized volatility, that is, after the price has already moved. Sasha is forward-looking: she reads risk from off-chain signals and prices the pool before the danger shows up in price.
How often does the fee change?
Up to once every 2 hours, on the autonomous keeper cron. In practice the fee only changes when Sasha's signal reading actually moves. If she stays at neutral, the keeper still re-pushes for liveness so the "oracle live" pill stays green.
What happens if Sasha goes silent?
The Oracle has a built-in 6-hour staleness threshold. If no push lands within 6 hours, isStale() returns true and the hook automatically falls back to 0.30% (DEFAULT_FEE). LPs are never exposed to a dead oracle.
Can I trade on this pool?
Yes. The pool is live on X Layer with real liquidity. Pair is USDC.e / WOKB, dynamic-fee flag, tickSpacing 60. Trade through Uniswap's canonical Universal Router on X Layer at 0xda00ae15…956d3 (Permit2 required for ERC20s).
How is this different from existing dynamic-fee hooks?
Two differences. (1) AI agent as the oracle: not a formula, not a TWAP, an autonomous agent fusing five off-chain inputs into one risk read. (2) Forward-looking: by the time on-chain volatility shows up, LPs have already been hit. Sasha tries to price the risk before that happens.
Why X Layer?
X Layer is OKX's Ethereum L2. Gas is roughly 0.02 gwei. Each setFee push costs about 0.000002 OKB, which makes a 2-hour autonomous keeper economically viable. Built for the OKX Build X Hackathon.
Glossary
Hook (Uniswap v4)
A smart contract that customizes how a Uniswap pool behaves. Hooks can intercept swaps, liquidity changes, and initialization to add custom logic. Here, to set the fee dynamically.
Dynamic fee
A swap fee that can change over the pool's lifetime, rather than being fixed at deployment. Enabled in v4 via the DYNAMIC_FEE_FLAG on the pool and a hook implementing _getFee().
Pool
A Uniswap v4 trading venue for a token pair. Identified by a PoolKey: (currency0, currency1, fee, tickSpacing, hooks).
LP (Liquidity Provider)
Anyone who deposits both tokens into the pool. LPs earn the swap fee from every trade. A static fee can hurt LPs two ways: too low during volatility (arbs pick them off) and too high during calm (honest flow leaves).
Oracle
Here, the contract Sasha writes to with her current fee reading (SashaOracle.setFee). The hook reads it via currentFee() on every swap.
risk-on / neutral / risk-off
Sasha's three signal tiers. Risk-on (500 bps, 0.05%) attracts volume when conditions are calm. Neutral (3000 bps, 0.30%) is the standard state. Risk-off (10000 bps, 1.00%) protects LPs from informed flow when she reads danger.
bps (basis points)
1 bp = 0.01%. So 3000 bps = 0.30%. Uniswap v4 fees are stored in bps as a uint24.
Stale
State where the Oracle hasn't been updated in more than 6 hours. Triggers the hook's automatic fallback to 0.30%.
beforeSwap
Uniswap v4 hook callback that fires before a swap executes. Sasha's hook uses it (with OVERRIDE_FEE_FLAG) to return the current fee.
_getFee()
The function Sasha's hook implements. Extends OpenZeppelin's BaseOverrideFee. Returns the fee in bps, called from beforeSwap.
Hook permission bits
Uniswap v4 encodes a hook's permissions in the last 14 bits of its contract address. The hook must be deployed at an address whose bits exactly match its declared permissions. Mined via CREATE2 through Nick's Factory at 0x4e59b448…. This hook's last 14 bits are 0x1080 (afterInitialize + beforeSwap).
LVR (Loss vs Rebalancing)
The largest hidden cost of providing liquidity to an AMM. LPs effectively lose to arbitrageurs because the AMM price lags off-chain price. Dynamic fees are one defense against LVR.
Universal Router
Uniswap's canonical swap aggregator. On X Layer it's deployed at 0xda00ae15…956d3. Routes through v4 pools (including this one), with Permit2 for ERC20 approvals.
Permit2
Uniswap's universal token-approval contract at 0x000000…78BA3. ERC20s approve Permit2 once; Permit2 then grants per-spender, per-amount allowances to the Router for each swap.
Keeper
The cron job that runs push-signal-to-xlayer.js every 2 hours from a VPS, signing each setFee tx from Sasha's agent EOA. Defined at /etc/cron.d/sasha-oracle.

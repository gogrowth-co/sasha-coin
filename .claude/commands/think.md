# /think — Perspective Shift + Problem Resolution

Use mid-conversation when the current approach feels stuck, too narrow, or when you want a different angle before committing to a direction.

**Focus (optional):** $ARGUMENTS

If $ARGUMENTS is empty, use the most recent problem or decision from the conversation as the subject.

---

## Instructions

This is a perspective reset. Do not continue down the same path you were already on. Start fresh from the problem, not from your previous answer.

Work through each phase. Keep it tight — this is a thinking tool, not a report.

---

### Phase 1 — Step back and restate

In one sentence: what is actually being solved here?

Then challenge the frame:
- Are we solving the right problem, or the most visible one?
- What assumption is being treated as fixed that might not be?
- What would someone outside this context say we're really trying to do?

---

### Phase 2 — Generate 3 genuinely different approaches

Not variations of the same idea. Three distinct angles:

1. **Minimal / eliminate** — What if we removed the problem instead of solving it? What's the smallest intervention that changes the outcome?
2. **Invert** — What would make this worse? Work backwards from failure to find what success actually requires.
3. **Reframe the constraint** — Take the biggest blocker and treat it as the design principle. What solution does it point to?

---

### Phase 3 — Pick the strongest and stress-test it

Choose the most promising approach from Phase 2. Then:

- What has to be true for this to work?
- What's the fastest way to find out if it doesn't?
- Is this reversible if it's wrong?

---

### Output

**New framing:** one sentence restating the problem through the lens that emerged  
**Recommended path:** what to do differently  
**Why this beats the previous direction:** one short paragraph  
**First move:** one concrete next action
